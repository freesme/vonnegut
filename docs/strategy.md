# 涨停板打板策略详解

## 一、策略概述

这是一套面向 A 股市场的**涨停板打板 + 情绪博弈**短线策略，核心逻辑是：

1. 从全市场 5000+ 只股票中，筛选出与"涨停"或"跌停"相关的 5 类候选模式
2. 通过竞价数据验证当日开盘状态，过滤不符合预期的标的
3. 用 6 因子评分系统量化排序，选出最优标的
4. 根据市场趋势动态调整模式优先级
5. 三层卖出规则 + tick 级监控控制风险

策略运行周期为 T+0 日内决策、T+1 持有，典型持仓周期 1~3 天。

## 四、选股逻辑（5 类模式）

### 4.1 时间锚点

所有选股数据基于 `ctx.previous_date`（上一个已收盘交易日，记为 T-1），不使用未来数据。

选股使用最近 3 个交易日：`date`(T-1)、`date_1`(T-2)、`date_2`(T-3)。

### 4.2 初始股票池

从全 A 股出发，逐步过滤：

1. **排除科创板/北交所**：仅保留 60/00/30 开头的主板和创业板
2. **排除上市不足 50 天的新股**
3. **排除 ST / 停牌 / 退市股**

### 4.3 五类模式

#### 模式一：连板龙头（lblt）

**定义**：T-1 收盘涨停，且近 10 天内至少连续 2 天涨停。

**筛选条件**：
- 取连板数最高的股票（"最高板"）
- 排除近 5 日有 2 个以上一字板的标的（流动性风险）

**买入方式**：以涨停价排板买入。

**市场含义**：连板股是短期市场最强主线的标志，龙头效应带来的溢价空间大，但对择时要求极高。

#### 模式二：弱转强（reversal / rzq）

**定义**：T-1 盘中曾触及涨停价但收盘未封住（冲高回落），且 T-2 未收盘涨停。

**筛选条件**：
- 近 3 日累计涨幅 ≤ 28%（排除过热标的）
- 当日竞价开盘比（开盘价/昨收）在 0.98 ~ 1.09 之间

**市场含义**：昨日涨停未封代表多空分歧，若次日竞价仍维持强势开盘，说明多方重新占优，是"弱转强"的典型形态。

#### 模式三：一进二（gap_up / gk）

**定义**：T-1 收盘涨停（首板），且 T-2、T-3 均未曾涨停。确保是"第一个涨停板"。

**筛选条件**：
- 当日竞价开盘比在 1.00 ~ 1.06（平开或小幅高开）
- 当前价格 ≤ 47 元（避免高价股流动性不足）

**市场含义**：首板涨停后次日若能高开走强，有望实现"一进二"（第一个板进第二个板），是打板策略中概率较高的介入点。

#### 模式四：首板低开（gap_down / dk）

**定义**：T-1 收盘涨停，且 T-2 未曾涨停（首板）。

**筛选条件**：
- 20 日相对位置 ≤ 0.5（不在阶段性高位）
- 当日竞价开盘比在 0.955 ~ 0.97（低开 3%~4.5%）
- T-1 成交额 ≥ 1 亿（确保流动性）

**市场含义**：首板涨停后低开代表获利盘抛压，若低开幅度可控（3~4.5%），可能是洗盘而非反转，存在低吸反弹机会。

#### 模式五：反向首板低开（fxsbdk）

**定义**：T-1 收盘跌停。

**筛选条件**：
- 20 日相对位置 ≤ 0.5
- 当日竞价开盘比在 1.04 ~ 1.10（跌停后次日高开）

**市场含义**：跌停次日大幅高开，说明有大资金抄底介入，跌停可能是恐慌性错杀。

## 五、市场趋势与优先级

### 5.1 趋势判断

基于沪深 300 指数近 3 日涨幅划分：

| 3 日涨幅 | 趋势 |
|---|---|
| > 2% | `strong_up`（强势上涨） |
| > 0.5% | `up`（温和上涨） |
| > -1.5% | `flat`（震荡） |
| ≤ -1.5% | `down`（下跌） |

### 5.2 模式优先级

不同市场环境下，模式的优先级不同：

| 趋势 | 优先级排序（高→低） |
|---|---|
| `strong_up` | 连板 > 弱转强 > 一进二 > 反向首板 > 首板低开 |
| `up` | 一进二 > 连板 > 弱转强 > 反向首板 > 首板低开 |
| `flat` | 连板 > 弱转强 > 一进二 > 反向首板 > 首板低开 |
| `down` | 连板 > 反向首板 > 一进二 > 弱转强 > 首板低开 |

**设计逻辑**：
- 强势市场中情绪高涨，连板和弱转强的成功率最高
- 上涨市场中首板一进二的空间最大
- 下跌市场中只有极强的连板龙头能穿越周期，反向首板的错杀修复机会增加

### 5.3 空仓条件

当沪深 300 指数连续 2 日量能异常（成交量 > 5 日均量 × 2.5 或 < 5 日均量 × 0.4）时，触发全面空仓。

## 六、6 因子评分系统

总分 = F1 + F2 + F3 + F4 + F5 + F6，满分 40 分，默认阈值 14 分。

### F1：涨停因子（0~5 分）

| 条件 | 得分 |
|---|---|
| T-1 收盘涨停 | +3 |
| T-1 收盘 ≥ 涨停价 × 98% | +2 |
| T-1 收盘 ≥ 涨停价 × 95% | +1 |
| T-1 成交量 > T-2 × 1.5 | +2 |
| T-1 成交量 > T-2 × 1.2 | +1 |

### F2：技术因子（0~10 分）

| 条件 | 得分 |
|---|---|
| 近 10 日涨停数 ≥ 5 | +5 |
| 近 10 日涨停数 ≥ 4 | +3 |
| 近 10 日涨停数 ≥ 1 | +2 |
| MA5 > MA10 > MA20（多头排列） | +2 |
| RSI 在 30~70 之间（健康区间） | +2 |
| 20 日价格位置 ≥ 50%（相对强势） | +1 |

### F3：放量 MA 因子（0~5 分）

| 条件 | 得分 |
|---|---|
| MA5 上穿 MA10（金叉） | +3 |
| 收盘价 > MA5 | +1 |
| 近 5 日均量 > 前 5 日均量 × 1.5（放量） | +2 |

### F4：主线因子（0~5 分）

匹配热门概念板块数量 × 2，上限 5 分。热门概念来自策略运行期间的概念数据缓存。

### F5：情绪因子（0~5 分）

| 条件 | 得分 |
|---|---|
| 市场趋势 `strong_up` / `up` | +2 |
| 市场趋势 `flat` | +1 |
| 全市场量能比 ≥ 1.5 | +2 |
| 全市场量能比 ≥ 1.2 | +1 |
| 个股相对强度超大盘 2% | +1 |

### F6：主力资金因子（0~10 分）

基于近 5 日资金流数据，三个维度加权：

- **流入比例**（40% 权重）：净流入 / 总额 > 30% 得 20 分，> 10% 得 12 分
- **绝对流入**（20% 权重）：> 1 亿得 10 分，> 5000 万得 6 分
- **连续性**（40% 权重）：近 3 日全部净流入得 15 分，2/3 得 8 分

三项加权后上限 10 分。

### 评分后的额外过滤

| 模式 | 额外条件 |
|---|---|
| 连板龙头 | F4 > 0 或 F6 > 0 或 F1 ≥ 5（至少有主线/资金/涨停板强度之一） |
| 一进二 | 总分在 20~37 之间（过高可能是陷阱） |
| 弱转强 / 首板低开 / 反向首板 | 总分 ≥ 20 |

## 七、买入执行

### 7.1 仓位管理

- 最大持仓：5 只（`POSITION_LIMIT`）
- 单票上限：总资产的 30%（`MAX_SINGLE_POSITION`）
- 资金均分：可用资金 / 可买入数量
- 排除当前已涨停的股票（涨停板大概率无法成交）

### 7.2 买入价格参考

| 模式 | 买入区间 |
|---|---|
| 连板龙头 | 涨停价（排板） |
| 弱转强 | 昨收 × 0.98 ~ 昨收 × 1.09 |
| 一进二 | 昨收 × 1.00 ~ 昨收 × 1.06 |
| 首板低开 | 昨收 × 0.955 ~ 昨收 × 0.97 |
| 反向首板低开 | 昨收 × 1.04 ~ 昨收 × 1.10 |

### 7.3 周五特殊处理

- 周一~四：早盘 09:28:10 竞价阶段执行买入
- 周五：早盘仅筛选不买入，14:50 尾盘根据全天走势二次筛选后建仓
- 周五二次筛选提高评分阈值（下跌市 18 分，其他 16 分），并要求量能比达标

## 八、三层卖出规则

### Layer 1：竞价阶段卖出（09:28）

在集合竞价结束时检查持仓，满足以下条件立即发出卖出信号：

**策略 1 — 涨停低开快速卖出**

前提：T-1 收盘涨停。

| 条件 | 触发 |
|---|---|
| 低开 1%~3%，流通市值 50~100 亿 | 卖出 |
| 低开 1%~5%，PE < 20 | 卖出 |

**策略 2 — 放量长上影开盘卖出**

前提：T-1 出现放量长上影线（上影线 > 下影线 × 1.2 且 > 实体 × 1.5，量比 > 1.5）。

| 条件 | 触发 |
|---|---|
| 当日开盘涨幅 < 2% | 卖出 |

### Layer 2：每 5 分钟技术止损

盘中每 5 分钟检查持仓，基于分钟级技术指标：

| 条件组合 | 操作 |
|---|---|
| 波段卖信号 + 放量大跌 | 紧急止损 |
| 波段卖信号 + 亏损 ≥ 3% | 波段卖出 |
| 放量大跌（跌幅 ≥ 8%，量比 ≥ 1.5） | 放量大跌止损 |

**波段卖信号**：MA3 < MA7 且 RSI > 65（短均线下穿长均线 + RSI 超买回落）。

### Layer 3：定时策略卖出（互斥）

每只股票每次检查只触发一个卖出策略：

**上午策略（09:30~11:30）**

1. **月初不涨停时间止损**：10 日涨幅 > 80% + 月初第一天 + 未涨停 → 卖出
2. **低于昨收**：当前价 < 昨日收盘价 → 卖出

**下午策略（13:00~15:00）**

3. **固定止损**：亏损 ≥ 5% 或距涨停价回撤 ≥ 15% → 卖出
4. **MA5 均线**：当前价 < MA5 → 卖出

**全时段**

5. **量价顶背离**：最新 30 分钟 K 线创新高但量能 ≤ 最大量 × 50%，且回落 > 3% → 卖出

### Tick 级实时监控（独立线程）

在交易时间内以 3 秒间隔轮询持仓股：

| 信号 | 条件 | 操作 |
|---|---|---|
| 急跌止损 | 1 分钟内跌幅 ≥ 3% | 立即卖出 |
| 放量大跌 | 跌幅 ≥ 6% 且估算日量比 ≥ 1.5 | 立即卖出 |
| 涨停打开 | 曾封涨停后回落 ≥ 2% | 立即卖出 |
| 跌停保护 | 当前价 ≤ 跌停价 | 不发卖出信号 |

## 九、数据层设计

### 9.1 抽象接口

`DataProvider` 定义了策略所需的全部数据接口：

- `get_price` — 历史 K 线（日线/分钟线）
- `get_realtime_quotes` — 实时行情快照
- `get_valuation` — 估值数据（PE/PB/市值）
- `get_money_flow` — 资金流向
- `get_call_auction` — 集合竞价数据
- `get_security_info` — 证券基本信息
- `get_concept` — 概念板块归属

### 9.2 数据源路由

```
有 Tushare Token?
  ├── 是 → CompositeProvider
  │         ├── 历史行情 / 估值 / 资金流 → TushareProvider
  │         └── 实时行情 / 分钟线 / Tick → EastMoneyProvider
  └── 否 → EastMoneyProvider（全功能降级）
```

### 9.3 缓存策略

所有历史数据写入 SQLite（`data_store/cache.db`），表结构：

- `daily_price` — 日 K 线 + 涨跌停价（按 code + trade_date 主键）
- `valuation` — 估值数据
- `security_info` — 证券信息

`prefetch_daily_date` 可批量预加载某日全市场日线（约 5000 条），避免逐只查询。缓存校验要求 ≥ 500 条记录才视为已完成批量预加载。

## 十、风控总结

| 层级 | 时机 | 手段 |
|---|---|---|
| 事前 | 选股阶段 | 大盘异常空仓、ST/停牌过滤、新股过滤、科创北交排除 |
| 事前 | 评分阶段 | 6 因子量化评估、模式特定阈值 |
| 事中 | 买入阶段 | 仓位上限 5 只、单票 ≤ 30%、排除已涨停标的 |
| 事后 | Layer1 (09:28) | 涨停低开 / 放量长上影 → 竞价卖出 |
| 事后 | Layer2 (每 5 分钟) | 波段卖 + 放量大跌 → 技术止损 |
| 事后 | Layer3 (定时) | 固定止损 5% / 涨停回撤 15% / MA5 / 量价背离 |
| 事后 | Tick 级 (3 秒) | 急跌 / 放量大跌 / 涨停打开 → 即时止损 |
| 全局 | 周五 | 早盘不买，尾盘二次确认后建仓（规避周末不确定性） |

## 十二、CLI 使用

```bash
# 执行完整选股（含买卖点分析）
python scan.py

# 仅对已有候选池重新评分
python scan.py --score-only

# 选股后推送通知
python scan.py --notify

# 自定义最低评分
python scan.py --min-score 16
```

输出包含：
1. 汇总表格（代码、名称、模式、6 因子分项得分）
2. 每只股票的买卖点详细分析块（买入区间、4 层止损价位、MA5 参考线）
